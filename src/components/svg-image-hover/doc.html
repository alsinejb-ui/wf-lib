svg-image-hover

Effet de survol sur des cartes : animation SVG stroke-dashoffset et apparition des mots du titre (GSAP). Dans Webflow : CSS et JS via Custom Code (liens wf-lib), puis structure 100 % native (collage Xscp ou création dans le Designer) — pas d’Embed HTML pour porter la structure.

Étape 1 — CSS & JS (Custom Code du site)

Les styles du composant viennent de wf-lib.css (pas d’Embed pour du CSS). Ajoutez le lien dans le <head> et le module JS avant </body> — le script initialise automatiquement les [data-component].

Étape 2 — Structure native dans le Designer

Collez une structure déjà construite (divs, classes card-img, svg-stroke, SVG inline dans des blocs natifs, attributs data-*) via le JSON Webflow ci-dessous, ou recréez la hiérarchie à la main en vous calant sur l’onglet Code (référence HTML uniquement).

Placez webflow-structure.xscp.json dans ce dossier (export depuis le Designer), ou collez le JSON après une copie dans le Designer.

Le collage Cmd+V / Ctrl+V sur le canvas recrée les éléments natifs et leurs classes — le CSS wf-lib (étape 1) s’applique alors comme sur la démo.

Designer — rappel (sans Embed)

  1. Section ou div racine : attribut data-component = svg-image-hover
  2. Chaque carte : data-svg-hover-card (vide), tabindex="0"
  3. Titre : data-svg-hover-title sur le heading
  4. SVG : blocs avec classes svg-stroke / svg-stroke-1 / svg-stroke-2 et <path> dans le DOM (comme dans un export Xscp) — une simple balise Image vers un fichier .svg ne suffit pas au script pour animer les traits.
