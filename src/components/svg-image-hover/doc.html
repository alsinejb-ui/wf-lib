svg-image-hover

Effet de survol sur des cartes : animation SVG stroke-dashoffset et apparition des mots du titre (GSAP). Dans Webflow : CSS et JS via Custom Code (liens wf-lib), puis structure 100 % native (collage Xscp ou création dans le Designer) — pas d’Embed HTML pour porter la structure. Pour Webflow, utilisez le script classique wf-lib.global.js (pas type="module") — voir le guide pas à pas dans l’onglet Webflow.

Guide pas à pas (sans expérience dev)

  1. Vérifiez le fichier sur GitHub. Le dépôt alsinejb-ui/wf-lib → dossier dist doit contenir wf-lib.global.js (script classique) et wf-lib.css. Sinon, demandez à quelqu’un qui a le projet de lancer npm run build puis d’envoyer ces fichiers sur GitHub (branche main).
  2. Ouvrez Webflow → votre site → menu Site settings (roue dentée) ou Project settings.
  3. Allez dans Custom code (ou équivalent : code global du site).
  4. Activez Run custom code in Preview (exécuter le custom code en mode Preview / commentaires) si Webflow le propose dans cette section — sinon le canvas *.canvas.webflow.com n’appliquera pas vos scripts. Réf. Aide Webflow — external resources in previews.
  5. Dans le champ Head code, collez le bloc « HTML — <head> » ci-dessous (le <link> vers wf-lib.css), puis enregistrez.
  6. Dans le champ Footer code (souvent « avant </body> »), collez le bloc « HTML — avant </body> » qui pointe vers wf-lib.global.js — sans type="module". Enregistrez.
  7. Publiez le site (bouton Publish). Attendez la fin du déploiement.
  8. Testez dans un vrai navigateur (Chrome, Safari) : ouvrez l’URL publiée du site (ex. https://votre-projet.webflow.io ou votre domaine personnalisé). Ne vous fiez pas seulement à l’aperçu dans le Designer : l’erreur CORS peut apparaître dans le Designer même si le site publié fonctionne, ou l’inverse.
  9. Vérifiez la page : la structure avec data-component="svg-image-hover" doit être sur la page (ou le symbole utilisé sur la page). Sans ça, rien ne s’anime.
  10. Si ça ne charge toujours pas : dans Site settings, regardez s’il y a une Content Security Policy personnalisée qui bloque cdn.jsdelivr.net. Sinon, il faudra héberger les fichiers .css / .js sur votre propre domaine (aide d’un dev ou hébergeur).

Canvas Webflow, CORS et wf-lib

En preview / commentaires, le site s’affiche sur {shortName}.canvas.webflow.com. Les ressources externes doivent pouvoir être chargées depuis ce domaine (voir Supporting custom code in preview mode et Supporting external resources in previews).

Le bundle wf-lib ne vérifie pas le hostname et ne bloque pas le canvas : il est compatible avec la doc Webflow. Des helpers sont exposés pour votre code custom (API, postMessage, etc.) : WFLib.isWebflowCanvasPreview(), WFLib.isWebflowHostedDomain(), WFLib.isWebflowAllowedOrigin(origin) (global IIFE), ou imports nommés depuis wf-lib.js en module ES.

Les snippets ci-dessous utilisent crossorigin="anonymous" sur le CSS et le JS pour un chargement CORS explicite, comme recommandé pour les ressources hébergées hors Webflow en preview.

Pourquoi wf-lib.global.js plutôt que type="module" ?

Webflow signale souvent « Unable to load external script » pour les modules ES depuis un CDN tiers. Le script classique wf-lib.global.js avec defer est en général mieux toléré sur l’URL publiée et en preview.

Si le script ne charge toujours pas :

  • Vérifier Run custom code in Preview (voir guide ci-dessus) et la CSP du projet (cdn.jsdelivr.net).
  • Héberger wf-lib.css et wf-lib.global.js sur votre domaine si le CDN refuse l’origine *.canvas.webflow.com (hors contrôle du code wf-lib).

Designer — rappel (sans Embed)

  1. Section ou div racine : attribut data-component = svg-image-hover
  2. Chaque carte : data-svg-hover-card (vide), tabindex="0"
  3. Titre : data-svg-hover-title sur le heading
  4. SVG : blocs avec classes svg-stroke / svg-stroke-1 / svg-stroke-2 et <path> dans le DOM (comme dans un export Xscp) — une simple balise Image vers un fichier .svg ne suffit pas au script pour animer les traits.
